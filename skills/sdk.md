## Scope naming convention

**Always** `x_<vendorprefix>_<appname>` — never `x_snc_`.

| PDI | Vendor prefix | Example scope |
|-----|--------------|---------------|
| dev390976.service-now.com | **9274** | `x_9274_axm_lite` |

The prefix is not `snc`, not the full instance name, not `dev` + number. It is the vendor prefix configured in the instance. When in doubt, check an existing scoped app on the PDI: System Applications → Applications → pick any scoped app and read its scope field.

## Project layout

The Fluent SDK project scaffolds **at the directory where `snc init` is run**. For axiom-lite, that is the repo root:

```
axiom-lite/           ← repo root = project root
  now.config.json     ← scope, scopeId, app name
  package.json        ← build/deploy/transform/types scripts
  src/
    fluent/           ← .now.ts Fluent source files
    server/           ← TypeScript server-side scripts
  @types/             ← generated by npm run types
```

**There is no `app/` subdirectory.** Run all commands from the repo root.

## `snc` alias — global install path

The `snc` alias resolves to:
```
node /opt/homebrew/lib/node_modules/@servicenow/sdk/bin/index.js
```

NVM's `npm install -g` goes to `~/.nvm/versions/node/vX/lib/node_modules/` — **not** the same path. To upgrade the SDK that `snc` uses:

```bash
npm install -g @servicenow/sdk@<version> --prefix /opt/homebrew
snc --version   # verify
```

## App creation and first deploy

```bash
# 1. Scaffold locally AND create app on PDI in one step
snc init \
  --appName  "My App" \
  --packageName "my-app" \
  --scopeName "x_9274_my_app" \
  --template "typescript.basic" \
  --auth dev390976

# 2. Install dependencies
npm install

# 3. Build + deploy (from repo root)
npm run build && npm run deploy
```

`snc init --auth <profile>` creates the app on the PDI at init time. If the app already exists with the same scope, subsequent deploys update it. If the scope in `now.config.json` doesn't match any app on the PDI, deploy fails with `application was null`.

### Template names (SDK 4.6.0)

| Context | Template flag value |
|---------|-------------------|
| New project | `typescript.basic`, `typescript.react`, `typescript.vue`, `javascript.basic`, `javascript.react` |
| Existing project (apply partial) | `partial.typescript.basic`, `partial.typescript.react`, `partial.typescript.vue`, `partial.javascript.basic`, `partial.javascript.react`, `partial.typescript.react-devserver` |

## Build loop (every component)

```bash
# From repo root — always
npm run build           # compile Fluent source → deployable package; read ALL output
npm run deploy          # install/update on PDI; read ALL output
npm run types           # refresh @types after any new custom table deploy
npm run transform       # pull PDI GUI changes back to source
```

Never commit without a clean build AND a validated deploy.

## Seed data — Record() with installMethod

Do **not** use Fix Scripts for seed data in a Fluent SDK project. Use the `Record()` API with `installMethod: 'first install'` — it's idempotent by `$id`.

```typescript
import { Record } from '@servicenow/sdk/core'

Record({
    $id: Now.ID['my-seed-record'],
    $meta: { installMethod: 'first install' },
    table: 'x_9274_myapp_my_table',
    data: {
        name: 'My Value',
        active: true,
    },
})
```

- `'first install'` → packed into `unload/` — loaded on every install, idempotent via `$id`
- `'demo'` → packed into `unload.demo/` — loaded only when demo data is requested
- No `$meta` → always installed

## Now.ID keys

Keys in `src/fluent/generated/keys.ts` are **auto-generated on first build** when a new `Now.ID['key-name']` is encountered. Do not edit `keys.ts` by hand. Commit it — it is the source of truth for record identity across deploys.

## Error reference

| Error | Cause | Fix |
|-------|-------|-----|
| `application was null` | App with this scope doesn't exist on PDI | Scope in `now.config.json` must match a real scoped app. Re-run `snc init --auth <profile>` with the correct `--scopeName` to create it. |
| `Could not find package.json` | Wrong working directory | Run from repo root (where `now.config.json` lives) |
| `Invalid scope` | scopeName mismatch | Check `now.config.json` scope vs the app in AES — never guess the vendor prefix |
| `Cannot find module '@servicenow/sdk'` | Missing deps or wrong SDK path | `npm install` then check `snc --version` matches `package.json` devDependency |
| `fetch failed` on `--reinstall` | Network/auth issue | Retry without `--reinstall`; check `snc auth --list` for active profile |
| Custom table not in trigger types | Stale type defs | `npm run types` after new table deploy, then `npm run build` |
| `active is not valid in FlowConfigProps` | Unsupported field | Remove `active` — activate in Flow Designer after deploy |
| `must be inside wfa.dataPill call` | Direct field access in flow | Wrap: `wfa.dataPill(expr, 'type')` |
